# Source Coverage

This is the operator view for deciding what to improve next. It is not a promise that every public
DC data source belongs in the release.

Tiers:

- Tier 0: needed for a credible first civic-structure release.
- Tier 1: high-value next improvement.
- Tier 2: useful completeness expansion.
- Tier 3: future or out of scope for now.

Current stress-test baseline: fresh workbench on June 1, 2026 fetched all 13 configured sources,
accepted 331 safe entity candidates, accepted a guarded Tier 0 relationship slice, and built a clean
release. The compact release had 330 accepted entities, 18 accepted relationships, 30 datasets, and
105 pending legal refs; the remaining review queue was still large.

## Tier 0 Gaps

1. Review and normalize branch/explicit-oversight relationship families beyond the first accepted
   governing-agency and committee `part_of` slice without claiming relationship completeness.
2. Review remaining placeholders created by accepted Open DC/MOTA relationships, especially
   role/person/organization endpoints that should not be treated as agencies.
3. Review the highest-confidence legal refs attached to Tier 0 entities, leaving ambiguous legal
   refs pending or deferred with visible status.
4. Decide whether Tier 0 needs a Board of Ethics / Open Government source lane before first usable
   release, or whether `dcgis.agencies`, Open DC, Council, and legal entrypoints are enough for the
   first release claim.

Next lane recommendation: review the remaining Tier 0 relationship families that still require
judgment, especially DCGIS branch rows and explicit Council oversight rows, while keeping Open DC
legal-authority placeholders deferred until legal-ref entities have their own review path.
